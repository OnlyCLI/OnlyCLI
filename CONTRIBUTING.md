# Contributing to OnlyCLI

Thank you for your interest in contributing to OnlyCLI. This guide explains how to set up your development environment and submit changes.

## Prerequisites

- Go 1.22 or later
- [golangci-lint](https://golangci-lint.run/welcome/install/)
- GNU Make

## Getting Started

```bash
# Fork and clone the repository
git clone https://github.com/<your-username>/onlycli.git
cd onlycli

# Install dependencies
go mod download

# Run all tests
make test

# Run the full pre-commit check
make check
```

## Project Structure

```
cmd/onlycli/          CLI entry point (Cobra commands)
internal/
  model/              Intermediate representation (IR) structs and naming utilities
  parser/             OpenAPI spec parsing via libopenapi
  codegen/            Code generation engine
    templates/        Go text/template files (.tmpl)
  testdata/           Test fixtures (petstore.yaml, github_subset.yaml)
integration_test.go   Integration tests
examples/             Generated CLI examples
```

## Development Workflow

1. **Create a branch** from `main`:
   ```bash
   git checkout -b feat/my-feature
   ```

2. **Make changes.** Code lives primarily in `internal/`. Templates for generated code are in `internal/codegen/templates/`.

3. **Write tests.** Every package has unit tests (`*_test.go`). Integration tests live at the repo root in `integration_test.go`. New features must include tests.

4. **Run the full check** before submitting:
   ```bash
   make check
   ```
   This runs `fmt`, `vet`, `lint`, and `test`.

5. **Commit** using [conventional commit](https://www.conventionalcommits.org/) messages:
   - `feat: add support for header parameters`
   - `fix: handle empty operationId gracefully`
   - `docs: update README with new flag`
   - `test: add edge case for multi-tag operations`
   - `chore: bump libopenapi to v0.35.0`

6. **Open a pull request** against `main`. Keep PRs focused and small.

## Code Style

- Follow standard Go formatting (`gofmt`). The linter enforces this.
- Generated code files must include the header: `// Code generated by onlycli; DO NOT EDIT.`
- Avoid adding dependencies unless necessary.
- Do not add comments that simply restate what the code does.

## Reporting Bugs

Use the [bug report template](https://github.com/onlycli/onlycli/issues/new?template=bug_report.md). Include your OS, Go version, onlycli version, and the OpenAPI spec that triggered the issue (if applicable).

## Feature Requests

Use the [feature request template](https://github.com/onlycli/onlycli/issues/new?template=feature_request.md). Describe the problem you are trying to solve and your proposed solution.

## Code Review

Maintainers review PRs for:

- Correctness and test coverage
- Consistency with existing patterns
- Clean commit history (squash if needed)
- No unnecessary dependency additions
- Documentation updates where relevant
