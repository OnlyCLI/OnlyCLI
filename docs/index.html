Why OnlyCLI?

Your spec is the source of truth. OnlyCLI turns it into a tool you can script, ship, and hand to agents—without paying a runtime tax every time something runs.

Zero Runtime

No interpreter, no VM. Pure compiled Go binary. 50× faster cold start than MCP tools.

Full API Coverage

Every endpoint becomes a CLI command. Auto-generated from your OpenAPI spec.

LLM Agent Ready

Native CLI that any AI agent can call directly. No MCP server needed.

Everything in one binary

Flags map cleanly to HTTP—output, auth, pagination, and payloads—so operators and automation share the same interface.

fmt

Multi-format output (json, yaml, table, csv, pretty)

Pick the shape that fits the next pipe: machine JSON, diff-friendly YAML, skim-friendly tables, or spreadsheets via CSV—without swapping tools.

--t

GJSON data transformation (--transform)

Project fields and arrays with GJSON paths on the command line. Skip spinning up jq when a single flag answers the question.

pg

Auto-pagination (--page-limit)

Walk Link: rel="next" headers and common cursor patterns until your cap—one invocation, complete lists.

@

@file input syntax

Pass large JSON bodies from disk with @path so shells stay readable and CI jobs stay repeatable.

OAuth2

OAuth2 authentication (device flow + client credentials)

Interactive device login for laptops; client credentials for runners. Tokens land where your security model expects them.

prof

Multi-profile configuration

Isolate staging, production, and personal tokens. Switch context with a profile flag instead of re-exporting secrets in every shell.

H2

HTTP/2 + gzip compression

Modern transport defaults: multiplexed requests where the server supports HTTP/2, compression when it helps latency.

retry

Retry with exponential backoff

Transient 429s and 5xx responses get bounded retries with backoff and jitter—tune with --max-retries for flaky networks.

dry

Dry-run mode

Print the method, URL, headers, and body that would be sent—ideal for reviews, docs screenshots, and teaching the CLI safely.

TAB

Tab completion with enum values

OpenAPI enums surface as completion candidates so shells suggest valid values instead of forcing guess-and-check.

a.b

Nested body field flags (--name.first)

Address nested JSON with dotted flags—structured request bodies without heredocs or temporary files.

{% raw %}{{ . }}{% endraw %}

Go template output (--template)

When paths are not enough, render responses with Go templates—custom summaries, one-line reports, or ad-hoc exporters.

Real-world: GitHub from the official spec

Point onlycli generate at GitHub’s published OpenAPI bundle, compile once, and drive repos and issues from the shell—or from an agent that already knows how to execute commands.

# Generate a GitHub CLI from the official spec
$ onlycli generate \
    --spec https://raw.githubusercontent.com/github/rest-api-description/main/descriptions/api.github.com/api.github.com.yaml \
    --name github --auth bearer --out ./github-cli

# Build it
$ cd github-cli && go mod tidy && go build -o gh .

# Use it
$ export GITHUB_TOKEN=ghp_xxx
$ ./gh repos list-for-user --username octocat --format table
$ ./gh issues create --owner myorg --repo myrepo --title "Bug fix" --body "Details..."
$ ./gh repos list-for-user --username octocat --transform "#.name"

Get started in 60 seconds

Install the generator, point it at your spec, build, and run --help. That is the whole loop.

# Install
go install github.com/onlycli/onlycli/cmd/onlycli@latest

# Generate your CLI
onlycli generate --spec your-api.yaml --name your-api --out ./cli

# Build & run
cd cli && go mod tidy && go build -o your-api .
./your-api --help
