Why OnlyCLI?

Complete coverage, zero effort

Every endpoint in your OpenAPI spec becomes a CLI command. Path parameters, query strings, request bodies -- all mapped to typed --flags automatically.

One binary, ship anywhere

Compiled Go binary. No interpreter, no VM, no node_modules. Copy it to a server, a container, or a colleague's laptop. It just runs.

AI agents use it natively

LLM agents call --help to discover commands, execute them, read stdout. No MCP server, no schema injection, no token waste.

Everything you need, built in

Not just commands and flags. Generated CLIs ship with the features you'd build yourself -- so you don't have to.

fmt

7 output formats

--format json|yaml|table|csv|pretty|jsonl|raw. Pick the shape that fits the next pipe -- human-readable tables, machine JSON, or diff-friendly YAML.

#.name

GJSON transforms

--transform "#.full_name" projects fields inline. Skip spinning up jq when a single flag answers the question.

pg

Auto-pagination

--page-limit 10 follows Link: rel="next" headers automatically. One invocation, complete result sets.

@

File & stdin input

--data @payload.json or --data @- for piped input. Keep shell commands readable and CI jobs repeatable.

OAuth2

OAuth2 built in

Device flow for interactive login. Client credentials for CI. Tokens saved per profile, not per env var.

prof

Multi-profile config

config set staging.token xxx then --profile staging. Separate dev, staging, and production without re-exporting secrets.

retry

Retry with backoff

--max-retries 3 handles transient 429s and 5xx with exponential backoff. Respects Retry-After headers.

dry

Dry-run mode

--dry-run prints method, URL, headers, and body without sending. Debug safely, document accurately.

TAB

Smart shell completion

Bash, zsh, fish, PowerShell. OpenAPI enums surface as completion candidates -- your shell suggests valid values.

{% raw %}{{ }}{% endraw %}

Go template output

--template '{% raw %}{{.login}}{% endraw %}' when you need custom formatting beyond field projection.

a.b

Nested body flags

--name.first John builds nested JSON. Structured request bodies without heredocs.

H2

HTTP/2 + compression

Modern transport defaults. Multiplexed requests, automatic gzip decompression.

Real-world: GitHub REST API

The full GitHub API spec has 1,107 operations across 43 groups. OnlyCLI generates a complete CLI in under 5 seconds.

# Generate from the official GitHub spec
$ onlycli generate \
    --spec https://raw.githubusercontent.com/github/rest-api-description/main/descriptions/api.github.com/api.github.com.yaml \
    --name github --auth bearer --out ./github-cli

# Build and authenticate
$ cd github-cli && go build -o gh .
$ export GITHUB_TOKEN=ghp_xxx

# List repos as a table
$ ./gh repos list-for-user --username octocat --format table

# Create an issue with flags
$ ./gh issues create --owner myorg --repo myrepo --title "Bug" --body "Details here"

# Search with field projection
$ ./gh search repos --q "language:go stars:>1000" --sort stars --transform "#.full_name"

# Paginate automatically
$ ./gh repos list-for-user --username octocat --page-limit 5 --format csv

Built for AI agents

MCP pushes full API schemas into every LLM prompt -- thousands of tokens per turn, whether the model uses them or not. OnlyCLI generates a binary that agents call like any other command.

# Agent discovers available commands
$ ./gh --help
$ ./gh repos --help

# Agent calls a specific operation
$ ./gh repos get --owner golang --repo go --format json

# Agent reads structured JSON from stdout
{"id":23096959,"name":"go","full_name":"golang/go",...}

Stable names. Predictable flags. Structured output. The agent learns the surface once, not on every turn.
